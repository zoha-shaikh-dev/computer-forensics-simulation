# computer-forensics-simulation
Basic simulation of employee database and hacker interface

A web-based simulation project designed to demonstrate fundamental concepts of computer forensics and web security. This project highlights that not only login credentials like email and password are important, but session tokens also play a critical role in controlling and securing user access.


🚀 Features

- Simulated employee database system
- Hacker interface to demonstrate unauthorized access scenarios
- Token-based session management for controlled access
- Data storage and handling using browser local storage
- Simple and user-friendly interface for easy understanding


🛠️ Technologies Used

- HTML
- CSS
- JavaScript


📌 How It Works

- The system simulates a basic login and data access environment
- User access is managed using session tokens stored in local storage
- The hacker interface and employee dashboard share the same local storage, allowing messages sent from the hacker interface to be visible on both interfaces
- This demonstrates how shared storage can lead to security risks if session data is not properly managed
- The project provides a simplified understanding of access control and data handling in web applications
